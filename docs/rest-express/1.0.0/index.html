rest-express – API de Gestión Jurídica (Rama Judicial)

Sistema de sincronización y gestión de actuaciones judiciales, carpetas y procesos para el bufete RS Asesor Jurídico, integrado con la API pública de la Rama Judicial de Colombia.

Descripción

Este proyecto es una API REST y conjunto de scripts de sincronización que:

  • Importa datos desde un archivo Excel maestro (general.xlsx) con información de ~500 carpetas jurídicas.
  • Consulta la API de Rama Judicial para obtener procesos y actuaciones judiciales.
  • Sincroniza con PostgreSQL vía Prisma ORM, manteniendo actualizada la base de datos con actuaciones, procesos, juzgados, demandas y deudores.
  • Envía notificaciones por Telegram cuando se detectan nuevas actuaciones relevantes.
  • Clasifica inteligentemente juzgados, tipos de proceso y patrones de actuaciones.

Base de Datos

Esquema Prisma (modelos principales)

  • Carpeta: Caso jurídico con número, nombre, categoría, llaveProceso
  • Demanda: Datos financieros y procesales de la demanda
  • Deudor: Información personal del demandado
  • Proceso: Proceso judicial vinculado a Rama Judicial
  • Actuacion: Movimiento procesal con fechas y consecutivo
  • Juzgado: Despacho normalizado con tipo, ciudad y URL
  • Nota: Anotaciones del abogado con fechas extraídas
  • Tarea: Tareas pendientes con subtareas

Requisitos

  • Node.js >= 18
  • Prisma ORM
  • Acceso a la API pública de la Rama Judicial

Instalación y configuración

  1. Instala las dependencias:
    pnpm install
# o npm install
  2. Configura la base de datos en prisma/schema.prisma y ejecuta las migraciones:
    npx prisma migrate dev
  3. Ajusta las variables de entorno si es necesario.

Uso

Ejecuta el script principal para sincronizar actuaciones:

node src/actuaciones.ts

Notas adicionales

  • El script implementa retardos automáticos entre peticiones para evitar bloqueos por parte de la API externa.
  • El manejo de errores es robusto: los fallos en la consulta de un proceso no detienen la sincronización global.
  • El código es fácilmente extensible para agregar nuevas fuentes de datos o modificar la lógica de actualización.
